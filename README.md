Étape 1: Identifier les Acteurs

Étape 4: Affiner et Valider

    Vérifiez la cohérence : Assurez-vous que chaque acteur est lié à au moins un cas d'utilisation.
    Simplifiez si nécessaire : Gardez le diagramme simple et lisible, surtout si c'est une première version.

Pour créer une API avec Node.js, Express, et MongoDB, et combiner certaines opérations d'ajout de plusieurs tables (ou collections) dans une seule fonction, il est logique de regrouper les opérations qui sont interconnectées ou dépendent les unes des autres. Voici les cas où la combinaison d'ajout pourrait être pertinente pour ton application de gestion des employés.
Combinaisons possibles d'opérations d'ajout :

    Ajout d'un nouvel employé et affectation initiale : Lorsque tu ajoutes un nouvel employé, il est courant de lui assigner un statut, une affectation initiale (dans un service et à un poste), et éventuellement de lui associer un diplôme ou une décision spécifique.
        Tables concernées : employé, statut_employé, affectation_employé, diplome, decision.
        Raison : Lorsqu'un nouvel employé est ajouté, il peut être utile d'insérer ses informations personnelles ainsi que sa situation initiale (statut, affectation, diplôme, décision) dans une seule transaction pour garantir la cohérence des données.

    Exemple d'ajout combiné :

    javascript

const ajouterNouvelEmploye = async (employeData, statutData, affectationData, diplomeData, decisionData) => {
  const session = await mongoose.startSession();
  session.startTransaction();
  try {
    const employe = new Employe(employeData);
    await employe.save({ session });
    
    const statut = new StatutEmploye({ ...statutData, employeId: employe._id });
    await statut.save({ session });
    
    const affectation = new AffectationEmploye({ ...affectationData, employeId: employe._id });
    await affectation.save({ session });
    
    if (diplomeData) {
      const diplome = new Diplome({ ...diplomeData, employeId: employe._id });
      await diplome.save({ session });
    }

    if (decisionData) {
      const decision = new Decision({ ...decisionData, employeId: employe._id });
      await decision.save({ session });
    }

    await session.commitTransaction();
    session.endSession();
    return employe;
  } catch (error) {
    await session.abortTransaction();
    session.endSession();
    throw error;
  }
};

Ajout d'un événement de carrière : Lorsqu'une mise à jour de carrière est effectuée, il est utile d'ajouter simultanément un historique de carrière et de mettre à jour son affectation (service ou poste) si nécessaire.

    Tables concernées : historique_carriere, affectation_employé.
    Raison : Lorsqu'un employé change de poste ou de service, l'historique de ce changement doit être enregistré, et l'affectation dans la table affectation_employé doit être mise à jour.

Exemple d'ajout combiné :

javascript

const ajouterEvenementCarriere = async (historiqueData, affectationData) => {
  const session = await mongoose.startSession();
  session.startTransaction();
  try {
    const historique = new HistoriqueCarriere(historiqueData);
    await historique.save({ session });
    
    if (affectationData) {
      await AffectationEmploye.updateOne(
        { employeId: historiqueData.employeId },
        { $set: affectationData },
        { session }
      );
    }

    await session.commitTransaction();
    session.endSession();
    return historique;
  } catch (error) {
    await session.abortTransaction();
    session.endSession();
    throw error;
  }
};

Ajout d'une décision et enregistrement dans le journal des actions : Lorsqu'une décision importante est prise concernant un employé (par exemple, promotion, mutation, licenciement), il est logique de lier cette opération à un journal des actions qui enregistre les actions de l'administrateur.

    Tables concernées : decision, journal_des_actions.
    Raison : La création d'une décision peut être accompagnée de la journalisation de cette action, en particulier dans les systèmes nécessitant une traçabilité des actions.

Exemple d'ajout combiné :

javascript

    const ajouterDecisionEtJournal = async (decisionData, journalData) => {
      const session = await mongoose.startSession();
      session.startTransaction();
      try {
        const decision = new Decision(decisionData);
        await decision.save({ session });

        const journal = new JournalDesActions(journalData);
        await journal.save({ session });

        await session.commitTransaction();
        session.endSession();
        return decision;
      } catch (error) {
        await session.abortTransaction();
        session.endSession();
        throw error;
      }
    };

D'autres combinaisons potentielles :

    Notification et Journalisation des actions : Chaque action majeure, telle que l'ajout ou la suppression d'un employé, pourrait entraîner l'envoi d'une notification à l'utilisateur et la journalisation dans le journal des actions.
    Modification de l'affectation et mise à jour de l'historique : Lorsqu'une affectation est mise à jour, tu peux simultanément mettre à jour l'historique de carrière pour que la modification soit enregistrée.

Conclusion :

Les combinaisons dépendront de tes besoins en automatisation et en cohérence des données. En général, combiner des opérations qui modifient plusieurs collections dépendantes dans une transaction est une bonne pratique pour garantir que les opérations restent atomiques et cohérentes.

## 📝 Description du projet

L'application **Gestion des Suivis de Carrière** a été conçue pour les besoins du **SRSP Atsimo Andrefana**. Elle permet de gérer, de manière centralisée et efficace, les informations relatives aux carrières des agents, leurs affectations, décisions, et compétences.
